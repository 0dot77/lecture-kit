저번 시간 복습: 진입점을 만든다

2주차의 핵심은 Claude에게 "내 프로젝트에서는 이렇게 행동해"라고 알려주는 구조를 만드는 것이었다.

파일 Claude Code Codex
CLAUDE.md 프로젝트의 헌법, 말투, 금지사항, 작업 규칙 같은 역할로 읽을 수 있지만, Codex에서는 AGENTS.md가 더 명시적인 표준 진입점
AGENTS.md 일반적으로는 Codex 계열에서 더 많이 쓰는 작업 지침 파일 현재 폴더와 하위 폴더 전체에 적용되는 운영 계약
SKILL.md /커맨드로 호출하는 특정 행동 정의 Codex에서도 로컬 스킬로 특정 워크플로우를 확장할 수 있음
hooks 이벤트가 일어날 때 자동으로 shell 커맨드를 실행 환경마다 다르지만, 핵심은 이벤트와 외부 행동을 연결하는 것

결국 중요한 것은 파일 이름이 아니라,
AI가 작업을 시작하기 전에 읽는 진입점을 설계하는 일이다.

빠른 점검

내 프로젝트 폴더에 아래 세 가지가 있는지 확인한다.

  • 나의 세계관과 작업 규칙이 담긴 CLAUDE.md 또는 AGENTS.md
  • 다시 쓰고 싶은 반복 행동이 정리된 SKILL.md 초안
  • 외부 도구와 연결하고 싶은 순간: 입력할 때, 저장할 때, 배포할 때
04 Agentic Loop

Claude Code의 동작 원리

Claude Code는 한 번에 정답을 내는 기계라기보다, 작은 루프를 반복하는 에이전트에 가깝다.

# Agentic Loop 1. gather context 현재 폴더, 열린 파일, 지침, 사용자의 요청을 읽는다 2. take action 파일을 만들거나 수정하고, 명령어를 실행하고, 도구를 호출한다 3. verify results 빌드, 테스트, 화면 확인, 에러 로그로 결과를 검증한다 4. loop 부족하면 다시 맥락을 모으고 수정한다
Artist View

작업실에서의 반복

재료를 본다. 손을 움직인다. 멀리서 다시 본다. 마음에 들 때까지 반복한다.

  • 스케치 보기
  • 한 획 추가하기
  • 전체 균형 확인하기
Engineering View

에이전틱 루프

컨텍스트 수집, 액션, 검증이 반복되면서 결과물이 점점 구체화된다.

  • 파일 읽기
  • 코드 수정
  • 테스트와 브라우저 확인
좋은 요청은 루프를 쉽게 만든다. "예쁘게 해줘"보다 "2주차 페이지 형식을 참고해서, week-03 강의 노트를 HTML 강의 페이지로 만들어줘"가 훨씬 좋은 이유다.
05 Claude Code Usage

유용한 Claude Code 사용법

오늘부터는 명령어와 기호를 외우는 것이 아니라, "어떤 맥락을 Claude에게 어떻게 건네는가"를 연습한다.

입력 용도 예시
/ 스킬 또는 내장 명령 호출 /help, /self-intro, /skill-creator
@ 특정 파일을 직접 지칭 @DESIGN.md, @index.html
Esc Esc 진행 중인 응답을 멈추거나 방향을 바꾸기 너무 길게 가거나 잘못 이해했을 때 중단
clear 현재 대화 화면 정리 새 작업을 시작할 때 시야 정리
compact 긴 대화를 요약해 이어가기 컨텍스트가 길어졌을 때 압축
resume / recap 이전 작업 흐름으로 돌아가기 어제 하던 프로젝트 이어가기
점점 통일되는 사용 패턴
  • 파일과 폴더를 직접 지칭한다
  • AI가 도구를 쓸 수 있게 권한과 맥락을 준다
  • 결과물을 눈으로 보고 다시 요청한다
  • 채팅창보다 데스크톱 앱, 코드 에디터, 브라우저 쪽으로 이동한다

파일을 지칭해서 요청하기

터미널에서 Claude Code를 실행한 뒤 아래처럼 요청한다.

@index.html 을 읽고, 이 페이지의 분위기에 맞춰 나의 자기소개 섹션을 하나 추가해줘. 완료 후 브라우저에서 확인할 수 있게 알려줘.

핵심은 "무엇을 참고할지"와 "어떻게 확인할지"를 같이 주는 것이다.

06 Custom Skill

나만의 스킬 만들기

스킬은 반복되는 창작 행동을 이름 붙인 것이다. 매번 긴 설명을 쓰는 대신, /내스킬로 호출한다.

.claude/skills/ └── my-skill/ └── SKILL.md

SKILL.md 기본 구조

--- name: my-skill description: 언제 이 스킬을 사용해야 하는지 한 문장으로 설명 --- # My Skill 이 스킬은 무엇을 만드는가? ## 입력 - 사용자가 제공해야 하는 정보 ## 실행 순서 1. 참고 파일을 읽는다 2. 결과물을 만든다 3. 브라우저나 빌드로 확인한다 ## 출력 - 어떤 파일을 만들고 어떻게 보여줄지
Skill Creator

스킬을 만드는 스킬

처음부터 완벽한 스킬을 쓰려 하지 말고, Skill Creator에게 내가 반복하고 싶은 행동을 설명한다.

  • 나는 무엇을 자주 만드는가?
  • 항상 참고하는 파일은 무엇인가?
  • 결과물은 어디에 생겨야 하는가?
Examples

유명한 스킬 읽기

좋은 스킬을 읽으면 "AI에게 일을 맡기는 문서"가 어떤 문법을 가지는지 보인다.

  • Frontend Skill: 화면을 어떻게 검증하는가
  • K-Skill: 특정 취향과 규칙을 어떻게 고정하는가
  • 내 스킬: 나의 반복 작업을 어떻게 짧게 호출하는가

오늘 만들 스킬 기획

아래 문장을 채워서 Skill Creator에게 전달한다.

나는 앞으로 Claude Code로 자주 ___를 만들고 싶다. 항상 참고해야 하는 내 취향/규칙은 ___이다. 결과물은 ___ 파일로 만들어졌으면 좋겠다. 완성 후에는 ___ 방식으로 확인하고 싶다. 이 작업을 /___ 라는 스킬로 만들 수 있게 SKILL.md를 설계해줘.
07 DESIGN.md

DESIGN.md — 시각 언어를 고정하기

CLAUDE.md가 프로젝트의 사고방식이라면, DESIGN.md는 프로젝트의 시각 언어다. 웹페이지를 만들 때 Claude가 매번 다른 분위기로 튀지 않도록 기준을 준다.

Google Stitch와 함께 쓰기

Stitch 같은 디자인 생성 도구에서 나온 화면을 그대로 끝내지 말고, 색, 레이아웃, 타이포그래피, 버튼 모양을 DESIGN.md로 번역한다. 그러면 Claude Code가 다음 화면도 같은 언어로 이어서 만들 수 있다.

# DESIGN.md ## 분위기 - 어둡고 조용한 작업실 - 기술적인 패널과 손으로 쓴 메모가 함께 있는 느낌 ## 색 - 배경: #0f1115 - 주요 텍스트: #f2f0e8 - 포인트: #d94f30 ## 레이아웃 - 첫 화면에서 프로젝트 제목이 크게 보일 것 - 섹션은 카드가 아니라 넓은 밴드로 나눌 것 - 모바일에서도 텍스트가 겹치지 않을 것 ## 금지 - 의미 없는 그라디언트 배경 금지 - 둥근 카드 남발 금지 - 사용법 설명 문구를 화면에 길게 쓰지 말 것
DESIGN.md는 취향의 안전장치다. "내가 원하는 분위기"를 매번 설명하지 않아도 되도록, AI가 읽는 디자인 헌법을 만든다.
08 Git & GitHub

GitHub 가입하고 밖으로 내보내기

오늘의 마지막 흐름은 내 컴퓨터 안의 파일을 웹 주소로 만드는 것이다. 여기서 Git은 변화 기록, GitHub는 그 기록을 올리는 공간, GitHub Pages는 정적 웹사이트 배포 기능이다.

개념 오늘의 설명
Git 작업의 스냅샷을 남기는 기록 장치
GitHub Git 기록을 인터넷에 올려두는 공간
add 이번 스냅샷에 포함할 파일을 고르는 행동
commit 고른 파일들을 하나의 기록으로 저장하는 행동
push 내 컴퓨터의 기록을 GitHub로 올리는 행동
Pages GitHub에 올라간 HTML을 웹사이트로 보여주는 기능

오늘 사용할 Git 흐름

작업 폴더에서 아래 순서만 기억한다.

# 1. 현재 상태 확인 git status # 2. 배포할 파일 선택 git add index.html style.css script.js # 3. 하나의 기록으로 저장 git commit -m "Publish my first vibe page" # 4. GitHub로 올리기 git push

  1. GitHub 계정 만들기

    github.com에 가입하고, 본인이 기억할 수 있는 사용자 이름을 정한다. 이 이름이 나중에 주소 일부가 된다.

  2. 새 repository 만들기

    내 프로젝트를 올릴 빈 저장소를 만든다. 처음에는 public으로 두는 것이 Pages 배포에 편하다.

  3. 파일 올리고 commit

    Claude Code와 함께 만든 HTML, CSS, JS 파일을 Git으로 기록한다.

  4. GitHub Pages 켜기

    Settings → Pages에서 branch를 선택하면 몇 분 뒤 웹 주소가 생긴다.

배포는 작품의 첫 관객을 만드는 일이다. 내 컴퓨터에서만 열리는 HTML이 아니라, 누군가에게 링크로 건넬 수 있는 작업이 된다.
09 Appendix

부록: npm과 localhost

오늘 모든 사람이 깊게 다루지는 않지만, 앞으로 웹 작업을 하다 보면 자주 만나는 두 단어다.

npm

자바스크립트 프로젝트에서 필요한 도구와 라이브러리를 설치하고 실행하는 명령어 묶음이다.

npm install npm run dev npm run build
localhost:3000

내 컴퓨터 안에서 임시로 켜진 웹사이트 주소다. 아직 인터넷에 배포된 것이 아니라, 내 컴퓨터에서만 열리는 작업실 주소라고 보면 된다.

http://localhost:3000 http://127.0.0.1:3000
언제 개발 서버가 필요한가?
  • 단순한 HTML 파일은 그냥 브라우저로 열어도 된다
  • 여러 파일을 import하거나, 프레임워크를 쓰거나, API 요청을 테스트하면 개발 서버가 필요하다
  • Claude Code가 npm run dev를 실행하면 보통 localhost 주소가 나온다
