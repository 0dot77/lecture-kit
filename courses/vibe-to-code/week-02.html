02 Environment Setup

환경 설정

오늘은 VS Code + 내장 터미널만 사용한다. 별도 프로그램 설치 없이 진행된다.

필요한 것
  • VS Code — code.visualstudio.com (이미 있으면 OK)
  • Claude Code — 이미 설치되어 있음 (1주차)
  • 터미널 열기 단축키: Ctrl + `

VS Code 화면 레이아웃 — 이렇게 배치하면 편하다:

┌────────────────────────────────────────────┐ │ 탐색기 │ 편집기 │ │ │ │ │ my-character/ │ CLAUDE.md 편집 중 │ │ ├── CLAUDE.md │ │ │ └── .claude/ │ │ │ ├── settings.json│ │ │ └── skills/ │ │ ├──────────────────────┴──────────────────────┤ │ 터미널 (Ctrl+`) │ │ $ claude │ └─────────────────────────────────────────────┘
팁: CLAUDE.md를 편집하면서 프리뷰 보기 — Ctrl+K 누른 뒤 V

첫 번째 세팅

터미널(Ctrl+`)을 열고 아래 명령어를 순서대로 실행한다:

# 1. 프로젝트 폴더 생성 mkdir my-character cd my-character # 2. .claude 폴더 구조 생성 mkdir -p .claude/skills/send touch CLAUDE.md .claude/settings.json .claude/skills/send/SKILL.md # 3. Claude Code 실행 claude

탐색기에 파일들이 보이면 준비 완료.

03 .claude 폴더란?

하네스를 입힌다는 것

Claude Code를 실행하면 Claude는 두 가지를 읽는다.

Claude가 읽는 것: 1. 당신이 지금 보내는 메시지 2. 프로젝트 폴더 안의 .claude/ ← 이게 하네스

메시지는 매번 달라지지만, 하네스는 Claude가 시작할 때부터 이미 읽혀 있다.

비유로 이해하기

비유 의미
헌법 이 프로젝트에서 Claude가 따르는 세계의 규칙
하네스 Claude가 이 역할을 할 수 있도록 입히는 장치
무대 대화가 일어나기 전에 이미 세팅된 공간

위치: 두 곳

하네스를 두는 위치에 따라 적용 범위가 달라진다:

# 전역 하네스 — 모든 프로젝트에 적용 ~/.claude/ └── CLAUDE.md # 프로젝트 하네스 — 이 폴더에서만 적용 my-project/ └── .claude/ ├── settings.json └── skills/
오늘은 프로젝트 하네스만 다룬다. 두 하네스가 충돌하면 프로젝트 하네스가 우선순위를 가진다.
04 내부 구조

3개의 파일이 전부다

.claude/ ├── CLAUDE.md ← 헌법: Claude가 이 세계에서 어떻게 존재하는가 ├── settings.json ← 운영 규칙: 어떤 도구를 쓸 수 있는가, 훅 └── skills/ └── [skill-name]/ └── SKILL.md ← 기술: /커맨드로 호출되는 행동

각 파일의 역할

파일 역할 언제 읽히는가
CLAUDE.md 세계관, 캐릭터, 규칙 정의 항상 — 모든 대화에
settings.json 권한 설정, 자동 반응(hooks) 세션 시작 시
SKILL.md 특정 행동 정의 /skill-name 호출 시
📁 /init 커맨드는 언제 쓰는가?

Claude Code 터미널에서 /init을 실행하면 Claude가 현재 폴더를 스캔해서 CLAUDE.md를 자동으로 채워준다.

/init 실행 ↓ Claude가 폴더 구조, 코드 파일 분석 ↓ CLAUDE.md에 프로젝트 설명 자동 작성

/init은 기존 코드 프로젝트에 Claude를 붙일 때 유용하다.
오늘처럼 세계관과 캐릭터를 처음부터 설계할 때는 직접 쓰는 게 맞다.

05 CLAUDE.md

CLAUDE.md — 세계의 헌법

Claude가 매 대화 시작 전에 읽는 파일.
"이 프로젝트에서 나는 어떤 존재인가"를 정의한다.

읽히는 순서

~/.claude/CLAUDE.md (전역) ↓ 프로젝트 루트의 CLAUDE.md ↓ 대화 시작

엔지니어링 관점으로 보면

Creative / Harness

CLAUDE.md = 세계관 설계도

Claude가 이 프로젝트에서 어떤 캐릭터인지, 어떤 규칙으로 말하는지, 이 세계의 법칙이 무엇인지 정의하는 자유 형식 문서.

  • 캐릭터의 정체성
  • 이 세계의 규칙
  • 응답 목소리와 톤
  • 해서는 안 되는 것
Engineering Parallel

System Prompt / Architecture Doc

소프트웨어의 시스템 프롬프트이자 아키텍처 문서와 같다. 코드가 실행되기 전에 항상 읽히는 전제 조건.

  • README.md — 프로젝트 목적과 규칙
  • System prompt — LLM 역할 설정
  • config.yaml — 런타임 기본값
  • .eslintrc — 코드 스타일 규칙

무엇을 쓰는가

자유 형식 마크다운. Claude가 읽을 수 있는 문장이면 무엇이든 된다.

쓸 수 있는 것들
  • 이 프로젝트가 무엇인지
  • Claude가 어떤 역할인지
  • 어떤 톤으로 말해야 하는지
  • 어떤 것을 하지 말아야 하는지
  • 이 세계의 규칙

예시 CLAUDE.md

# 2066서울 나는 기억 거래소의 문지기다. 이 세계에서 기억은 날씨처럼 흐른다. ## 규칙 - 진짜 이름을 묻지 않는다 - 모든 응답은 현재형으로 - 불확실한 것은 침묵한다 ## 나의 목소리 짧게 말한다. 확신이 없으면 말하지 않는다. "아마도"라는 단어는 쓰지 않는다.

나만의 CLAUDE.md 작성하기

아래 질문에 답하면서 CLAUDE.md 초안을 작성한다. 최소 3문장, 완성하려 하지 말 것.

  • 나는 이 세계에서 누구인가? (한 문장)
  • 이 세계의 규칙은 무엇인가? (2개 이내)
  • Claude는 어떤 목소리로 말해야 하는가?
막힐 때: Claude Code에서 "내 캐릭터 설정을 도와줘. 키워드: [내가 떠올린 단어들]" 이라고 입력해보자. Claude가 CLAUDE.md 초안을 써줄 것이다.
06 Skills

Skills — 커스텀 커맨드

특정 명령어(/skill-name)로 호출되는 행동.
Claude에게 "이 커맨드가 들어오면 이렇게 행동해"를 가르치는 파일이다.

파일 구조

.claude/skills/ └── send/ └── SKILL.md ← /send 입력 시 실행

SKILL.md 형식

# [skill 이름] [이 skill이 무엇을 하는가 — 1-2문장] ## 실행 순서 1. [첫 번째 행동] 2. [두 번째 행동] 3. [결과 출력 방식]

엔지니어링 관점으로 보면

Creative / Harness

SKILL.md = 캐릭터의 특기

특정 상황에서 캐릭터가 어떻게 행동할지를 절차적으로 정의한다. /send를 호출하면 캐릭터의 관점으로 선택지를 생성하고 서버에 전송한다.

  • 반복 행동의 자동화
  • 특정 시점의 특정 반응
  • 외부 시스템과의 연결
Engineering Parallel

Function / CLI Command

소프트웨어의 함수 또는 CLI 커맨드와 같다. 특정 입력(커맨드)을 받으면 정해진 순서대로 실행하는 루틴.

  • 함수(function) — 호출 시 실행
  • CLI 커맨드 — npm run build
  • API 엔드포인트 — POST /action
  • 스크립트 — bash deploy.sh

언제 쓰는가

  • 반복해서 하는 행동이 있을 때
  • 특정 순간에 특정 방식으로 행동해야 할 때
  • 다른 시스템(서버, API)과 통신해야 할 때

오늘의 핵심 Skill: /send

이야기의 선택 지점에서 캐릭터로서 선택지를 생성하고 서버에 전송한다.

# send 이야기의 선택 지점에서 내 캐릭터의 관점으로 선택지를 생성하고 서버에 전송한다. ## 실행 순서 1. 현재 이야기 컨텍스트를 서버에서 가져온다 curl http://[서버IP]:3000/context 2. 나의 캐릭터(이 CLAUDE.md의 주인)로서 이 순간 취할 행동이나 발언을 1개 만든다 - 20-40자 한국어 - 캐릭터의 본질과 목소리를 반영 - 추상적 태도가 아닌 구체적 행동 또는 발언 3. 만들어진 선택지를 서버에 전송한다 curl -X POST http://[서버IP]:3000/choice \ -H "Content-Type: application/json" \ -d '{"participantId":"[내ID]","characterName":"[캐릭터명]","text":"[선택지]"}' 4. "전송됨: [선택지 내용]" 출력

/send 설치하기

  • .claude/skills/send/SKILL.md 파일에 위의 내용을 복사한다
  • [서버IP], [내ID], [캐릭터명]을 실제 값으로 교체한다
  • Claude Code에서 /send 입력 → 어떤 선택지가 나오는지 확인
핵심 질문: /send의 결과가 달라지려면 무엇을 바꿔야 할까?
→ SKILL.md가 아니라 CLAUDE.md다. skill은 방법을 정의하고, CLAUDE.md가 캐릭터를 정의한다.
07 Hooks

Hooks — 자동 반응 장치

Claude Code의 특정 이벤트에 shell 커맨드를 자동으로 실행시키는 장치.
settings.json 안에 정의한다.

이벤트 종류

Hook 발동 시점
UserPromptSubmit 메시지를 입력할 때마다
PreToolUse Claude가 도구를 사용하기 직전
PostToolUse Claude가 도구를 사용한 직후
Stop Claude가 응답을 끝냈을 때

엔지니어링 관점으로 보면

Creative / Harness

Hooks = 무대의 감각기관

Claude가 무언가를 할 때마다 외부 세계가 반응한다. 내가 타이핑하면 무대의 캐릭터가 pulse하고, /send가 완료되면 골드 발광이 일어난다.

  • 행동 → 무대 반응
  • 실시간 신호 전송
  • 캐릭터 상태 동기화
Engineering Parallel

Event Listener / Middleware

소프트웨어의 이벤트 리스너, 미들웨어, 또는 CI/CD 트리거와 같다. 특정 이벤트 발생 시 자동으로 실행되는 코드.

  • addEventListener('click', fn)
  • Express middleware — req/res pipeline
  • GitHub Actions — on: push
  • Webpack — beforeCompile hook

기본 형식 (settings.json)

{ "permissions": { "allow": ["Bash(curl:*)"] }, "hooks": { "UserPromptSubmit": [ { "hooks": [{ "type": "command", "command": "echo '[$(date +%H:%M:%S)] 입력 감지'" }] } ], "PostToolUse": [ { "matcher": "Bash", "hooks": [{ "type": "command", "command": "echo '[$(date +%H:%M:%S)] 도구 사용 완료'" }] } ] } }
⚠️ 중요: permissions 설정 필수

permissions.allow에 "Bash(curl:*)"가 있어야 curl이 실행된다.
이게 빠지면 훅은 실행되지만 curl은 동작하지 않는다.

오늘 훅이 하는 일

메시지 입력 또는 /send 실행 시 → 서버에 신호 전송 → 무대의 캐릭터가 반응

{ "permissions": { "allow": ["Bash(curl:*)"] }, "hooks": { "UserPromptSubmit": [{ "hooks": [{ "type": "command", "command": "curl -s -X POST http://[서버IP]:3000/signal -d '{\"participantId\":\"[내ID]\",\"event\":\"thinking\"}' &" }] }], "PostToolUse": [{ "matcher": "Bash", "hooks": [{ "type": "command", "command": "curl -s -X POST http://[서버IP]:3000/signal -d '{\"participantId\":\"[내ID]\",\"event\":\"done\"}' &" }] }] } }

무대에서 보이는 것

  • 타이핑 시작 → 해당 캐릭터 스프라이트 pulse
  • /send 완료 → 캐릭터 골드 발광 + 선택지 패널 등장
  • 완료 → 캐릭터 대기 상태 복귀

settings.json 설정하기

  • 배포된 settings.json을 .claude/settings.json에 붙여넣는다
  • [내ID], [캐릭터명]을 본인 것으로 수정한다
  • Claude Code에서 아무 메시지나 입력 → 무대 화면에 내 캐릭터가 반응하는지 확인
08 이야기 무대

클라이맥스 — 이야기 무대

각자가 설계한 하네스가 하나의 공동 이야기를 만든다.

진행 흐름

  1. 세계관 하네스로 무대가 열린다

    공용 세계관 CLAUDE.md를 기반으로 첫 이야기 beat가 생성되어 화면에 흐른다. 프로젝터에 무대 배경과 캐릭터들이 등장한다.

  2. BLANK — 선택 지점

    이야기가 멈추고 선택 지점이 등장한다. 각자 Claude Code에서 /send가 활성화된다.

  3. 각자 /send 실행

    나의 CLAUDE.md (캐릭터 하네스)로 이 순간의 선택지가 생성된다. 서버에 전송되면 무대 패널에 선택지가 등장한다.

  4. 함께 선택

    화면에 모든 선택지가 모인다. 함께 읽고 투표 또는 토론으로 다음 방향을 선택한다.

  5. 이야기 분기 — 반복

    선택된 선택지로 다음 beat가 생성된다. 2단계부터 반복. 2–3 사이클.

관찰할 것

  • 같은 이야기를 받았는데 선택지가 왜 다른가?
  • 어떤 CLAUDE.md가 가장 독특한 선택지를 만들었는가?
  • 내 캐릭터의 목소리가 선택지에서 느껴지는가?
선택지의 '품질'보다 '다름'에 집중한다.
이 선택지가 왜 이 캐릭터에서 나왔는지 설명할 수 있으면 성공이다.
이야기가 이상하게 흘러가도 괜찮다 — 그게 그 캐릭터들이 모인 세계의 결과다.
