01

CLAUDE.md 설정

Project Instructions 완전 가이드

CLAUDE.md란?

CLAUDE.md는 Claude Code가 프로젝트별로 자동으로 읽는 설정 파일이다. 매 세션마다 반복되는 프로젝트 규칙, 컨벤션, 제약사항을 한 번만 작성하면 Claude가 지속적으로 준수한다.

파일 위치와 우선순위

CLAUDE.md는 여러 위치에 배치 가능하며, 더 구체적인 범위일수록 우선순위가 높다.

위치 용도 공유 범위
~/.claude/CLAUDE.md 개인 코딩 스타일 나만 (모든 프로젝트)
./CLAUDE.md 팀 프로젝트 규칙 팀 (소스 컨트롤)
.claude/rules/*.md 모듈화된 주제별 규칙 팀 (소스 컨트롤)
./CLAUDE.local.md 개인 프로젝트별 설정 나만 (현재 프로젝트)
{subdir}/CLAUDE.md 특정 모듈 전용 해당 디렉토리만

Claude Code는 작업 디렉토리에서 루트까지 상위 디렉토리를 재귀적으로 탐색하여 모든 CLAUDE.md 파일을 수집한다.

기본 구조

효과적인 CLAUDE.md는 4개 주요 영역으로 구성된다:

CLAUDE.md 기본 템플릿
# {프로젝트명}

## 프로젝트 개요
[1-3문장으로 프로젝트 설명]

## 기술 스택
[사용 기술, 버전, 중요 라이브러리]

## 코드 컨벤션
[언어 규칙, 포맷, 네이밍 규칙]

## 주의사항 / 절대 하지 말 것
[잘못하면 문제 발생하는 사항들]
실전 예시 — Next.js + TypeScript
CLAUDE.md
# E-Commerce Platform

## 프로젝트 개요
Next.js 14 (App Router) 기반 B2C 이커머스 플랫폼.
결제는 Stripe, 인증은 NextAuth.js, DB는 PostgreSQL + Prisma.

## 기술 스택
- Next.js 14.2, React 18, TypeScript 5.3
- Tailwind CSS 3.4 (CSS Modules 금지)
- Prisma ORM, PostgreSQL 15
- Vitest, ESLint + Prettier
- pnpm (npm/yarn 금지)

## 코드 컨벤션
- 컴포넌트: PascalCase, named export
- 훅: `use` 접두사, `hooks/` 디렉토리
- `type` 키워드 사용 (interface 지양)

## 절대 하지 말 것
- `any` 타입 사용 금지
- `console.log` 프로덕션 코드에 남기기 금지
- DB 쿼리 직접 작성 금지 — Prisma 필수
- `process.env` 직접 접근 금지 — `lib/env.ts` 통해서만
@ 임포트 구문

CLAUDE.md에서 다른 파일을 @ 접두사로 참조하면 해당 내용도 컨텍스트에 포함된다:

CLAUDE.md
# My Project

@README.md 를 참고해서 프로젝트 개요를 파악해.
@package.json 에서 사용 가능한 npm 스크립트 확인.

## 추가 지침
- git 워크플로우 @docs/git-instructions.md
형식 의미
@filename 현재 CLAUDE.md 기준 상대 경로
@path/to/file 현재 CLAUDE.md 기준 상대 경로
@~/path 홈 디렉토리 기준
.claude/rules/ — 모듈화된 규칙

대규모 프로젝트는 규칙을 여러 파일로 분리할 수 있다. .claude/rules/ 내 모든 .md 파일은 자동으로 로드된다.

디렉토리 구조
your-project/
├── .claude/
│   ├── CLAUDE.md
│   └── rules/
│       ├── code-style.md
│       ├── testing.md
│       ├── security.md
│       ├── frontend/
│       │   ├── react.md
│       │   └── styles.md
│       └── backend/
│           ├── api.md
│           └── database.md

YAML frontmatter로 특정 파일에만 적용되는 조건부 규칙도 만들 수 있다:

.claude/rules/api.md
---
paths:
  - "src/api/**/*.ts"
---

# API 개발 규칙
- 모든 API 엔드포인트에 입력 검증 포함
- 표준 에러 응답 형식 사용
- OpenAPI 문서화 주석 포함
CLAUDE.md 작성 팁 5가지
  1. 명령형으로 작성 — "TypeScript를 사용하는 프로젝트입니다" (X) → "TypeScript만 사용해. JavaScript 파일 생성 금지" (O)
  2. 금지사항 명시 — 금지사항을 명확히 하면 실수가 감소한다
  3. 구체적으로 작성 — "좋은 코드를 써달라"가 아니라 "함수 하나당 30줄 초과 시 분리"
  4. 파일 길이 관리 — 공식 권장: CLAUDE.md 파일당 200줄 이하
  5. 팀이 함께 관리 — CLAUDE.md는 코드 리뷰 대상. PR로 수정하고 함께 검토
빠른 시작

터미널에서 /init 커맨드를 실행하면 프로젝트를 분석해 CLAUDE.md를 자동 생성할 수 있다.

02

권한 모드

도구 실행 제어와 승인 시스템

6가지 권한 모드
모드 설명 사용 사례
default 첫 도구 사용 시 승인 요청 일반 개발
acceptEdits 파일 수정 자동 승인 신뢰할 수 있는 로컬 개발
plan 분석만, 수정/실행 불가 코드 탐색
auto 분류기가 자동 승인/차단 장시간 세션
dontAsk 사전 승인된 도구만, 나머지 자동 거부 제한적 자동화
bypassPermissions 모든 승인 건너뜀 격리된 컨테이너/VM 전용

세션 중 Shift+Tab 또는 Alt+M으로 실시간 전환 가능

승인 옵션 이해하기

default 모드에서 Claude가 승인을 요청할 때 선택할 수 있는 응답:

y (Yes) 이번 한 번 허용
n (No) 거부 — Claude가 대안을 시도
a (Always) 이 세션에서 이 유형을 자동 승인
d (Don't) 이 세션에서 이 유형을 항상 거부
권한 규칙 관리

/permissions 커맨드로 현재 규칙을 확인하고 관리할 수 있다. 평가 순서: deny → ask → allow (deny가 항상 우선)

settings.json
{
  "permissions": {
    "allow": [
      "Read",
      "Bash(npm run *)",
      "Edit(/src/**/*.ts)"
    ],
    "deny": [
      "Bash(rm *)",
      "Bash(git push *)"
    ]
  }
}
모드 선택 가이드
무엇을 하고 있나요?
├─ 새 코드베이스 탐색/이해
│  → Plan 모드 (읽기 전용)
├─ 빠른 로컬 개발
│  → acceptEdits 또는 default 모드
├─ 장시간 리팩토링, 승인 피로 줄이기
│  → Auto 모드 (분류기가 판단)
├─ 팀 프로젝트, 중요 코드
│  → Default 모드 + settings.json 규칙
├─ CI/CD, 자동화 스크립트
│  → Headless 모드 + allowedTools 지정
└─ 격리된 Docker/VM 테스트
   → bypassPermissions (주의!)
03

Git 연동

AI 기반 커밋, 리뷰, PR 워크플로우

Claude Code가 Git에서 읽는 것들
  • ● 현재 브랜치 이름
  • ● 최근 커밋 히스토리
  • ● Staged / Unstaged 변경사항
  • ● 파일 구조 및 수정 시간

이 컨텍스트 덕분에 최소한의 프롬프팅으로 코드 변경사항을 이해할 수 있다.

커밋 메시지 자동 생성
터미널
git add src/auth/login.ts tests/auth.test.ts
claude> 스테이징된 변경사항 보고 커밋 메시지 써줘

CLAUDE.md에 Conventional Commits 가이드라인을 추가하면 type(scope): description 형식을 강제할 수 있다.

AI 코드 리뷰
터미널
claude> main 브랜치와 비교해서 현재 변경사항 코드 리뷰해줘

보안 취약점, 성능 문제, 엣지 케이스, 테스트 커버리지 갭에 집중하여 리뷰한다.

PR 설명 자동 작성

Claude가 변경사항을 요약하여 PR description을 자동 생성한다. 테스트 요구사항, 중요 참고사항도 포함된다.

충돌 해결

Claude Code는 merge conflict 마커를 분석하고, 양쪽의 의도를 파악하여 해결안을 제안한다.

실전 워크플로우
# 1. 최신 코드 가져오기
git pull origin main

# 2. 피처 브랜치 생성
git checkout -b feat/user-auth

# 3. Claude와 함께 기능 구현
claude> JWT 인증 로직을 구현해줘

# 4. 변경사항 리뷰
claude> 지금까지 변경사항 코드 리뷰해줘

# 5. 커밋 메시지 생성
git add .
claude> 커밋 메시지 써줘

# 6. PR 설명 생성
claude> PR description 작성해줘
04

슬래시 커맨드

빠른 실행 단축키와 키보드 컨트롤

슬래시 커맨드란?

슬래시 커맨드(/)는 Claude Code 대화 중에 특정 동작을 빠르게 실행하는 단축키이다. 긴 문장을 타이핑하지 않고 /로 시작하는 짧은 명령으로 자주 하는 작업을 즉시 실행할 수 있다.

세션 관리
커맨드 설명
/clear 대화 기록 초기화 (별칭: /reset, /new)
/compact [지시] 대화 기록을 요약으로 압축
/rewind 이전 턴 되돌리기
/resume 이전 세션 이어가기
/fork [이름] 현재 대화에서 분기하여 새 세션 생성
/export 현재 대화를 파일 또는 클립보드로 내보내기
정보 확인
커맨드 설명
/help 사용 가능한 모든 커맨드 목록
/status 버전, 모델, 계정, 연결 상태
/cost 토큰 사용량 통계
/context 현재 컨텍스트 사용량 시각화
/doctor 설치 진단 (Node.js, API, 설정 검증)
코드 작업
커맨드 설명
/diff 인터랙티브 diff 뷰어
/copy 마지막 응답 클립보드 복사
/pr-comments GitHub PR 댓글 가져오기
/init 프로젝트 분석 후 CLAUDE.md 자동 생성
/memory CLAUDE.md 편집, 자동 메모리 관리
키보드 단축키
단축키 설명
Ctrl+C 현재 입력 또는 생성 취소
Ctrl+L 터미널 화면 클리어 (대화 기록은 유지)
Ctrl+B 실행 중인 작업을 백그라운드로 전환
Ctrl+V 클립보드에서 이미지 붙여넣기
Shift+Tab 권한 모드 전환
Esc + Esc 체크포인트로 되돌리기
Alt+P 모델 전환 (프롬프트 유지)
빠른 입력 접두사
/ 슬래시 커맨드 또는 스킬 호출
! Bash 모드 — Claude 거치지 않고 직접 실행, 결과는 컨텍스트에 추가
@ 파일 경로 자동완성
커스텀 슬래시 커맨드 만들기

자주 하는 작업을 커스텀 커맨드로 만들어두면 반복 입력을 줄일 수 있다.

.claude/commands/review.md
지금까지 변경된 코드를 리뷰해줘.
다음 항목을 중심으로:

1. 보안 취약점 (OWASP Top 10)
2. 성능 문제 (N+1 쿼리, 불필요한 반복)
3. 타입 안전성
4. 엣지 케이스 처리
5. 테스트 커버리지

각 항목마다 심각도(높음/중간/낮음)로 분류해서 알려줘.

이제 대화 중에 /review라고 입력하면 위 프롬프트가 자동 실행된다. $ARGUMENTS를 사용하면 실행 시 추가 인자를 전달할 수 있다.

자주 쓰는 커맨드 실전 활용
  • /clear — 새 작업 시작 시 이전 대화 노이즈 제거
  • /compact — 대화가 길어졌지만 초기화하기는 싫을 때 핵심만 유지
  • /cost — 토큰 사용량과 비용 확인
  • /init — 새 프로젝트에서 CLAUDE.md 자동 생성
  • /resume — 이전 세션을 이어서 작업
05

컨텍스트 관리

토큰 절약과 효율적인 세션 운영

컨텍스트 창이란?

Claude 모델이 한 세션에서 "기억"할 수 있는 텍스트 총량이다: 

┌─────────────────────────────────────────┐
│     컨텍스트 창 (기본 200K / 확장 1M 토큰)   │
│                                         │
│  [대화 히스토리] [코드] [파일 내용] [응답]  │
│                                         │
└─────────────────────────────────────────┘
  • ● 기본: 200,000 토큰 (200K)
  • ● Opus 4.6 / Sonnet 4.6: 최대 1,000,000 토큰 (1M) 확장 가능
  • ● 대략 한국어 약 10만 단어, 코드 파일 약 150개(50줄 기준)
컨텍스트가 가득 차면?

컨텍스트가 70-80%에 도달하면:

  1. 응답 속도 저하 — 처리할 텍스트가 많아질수록 느려짐
  2. 초반 내용 망각 — 초기 대화 정확도 하락
  3. 비용 증가 — 토큰이 많을수록 API 비용 증가
  4. 자동 압축 발동 — Claude Code가 오래된 도구 출력부터 제거
컨텍스트 관리 6가지 전략

전략 1: /compact로 압축

긴 대화를 초기화하지 않고 핵심만 유지한다. 커스텀 지시사항 추가 가능: /compact 코드 샘플에 집중해서 요약해줘

전략 2: /clear로 새 시작

완전히 다른 작업을 시작할 때 사용. CLAUDE.md 내용은 유지된다.

전략 3: 작업을 작게 분할

하나의 긴 세션보다 여러 짧은 세션이 효과적. 커밋 후 /clear.

전략 4: 파일 직접 참조

코드를 대화창에 붙여넣지 말고 src/payment/stripe.ts 파일 리뷰해줘처럼 경로를 지정한다.

전략 5: CLAUDE.md에 중요 정보 저장

세션 간 유지되어야 할 진행 상황, 중요 결정사항을 CLAUDE.md에 기록한다.

메모리 시스템

세션 간 기억 유지와 6계층 아키텍처

왜 메모리가 필요한가?

Claude Code는 세션이 끝나면 대화를 기억하지 못한다. 6계층 메모리 아키텍처를 통해 세션 간 연속성을 유지한다.

메모리 계층 (우선순위 순)
순위 계층 위치 / 설명
1 관리형 정책 Enterprise — 조직 전체 CLAUDE.md
2 사용자 메모리 ~/.claude/CLAUDE.md — 개인 전역 설정
3 프로젝트 메모리 ./CLAUDE.md — 팀 공유 (git 추적)
4 프로젝트 규칙 .claude/rules/ — 모듈화된 조건부 규칙
5 로컬 메모리 ./CLAUDE.local.md — 개인 프로젝트별 (git 무시)
6 자동 메모리 ~/.claude/projects/.../memory/ — Claude 자동 학습

더 구체적인 (좁은 범위의) 설정이 광범위한 설정보다 우선적용된다.

자동 메모리 (Auto Memory)

Claude가 자율적으로 발견한 패턴, 디버깅 인사이트, 사용자 선호도를 기록한다.

  • ● MEMORY.md의 첫 200줄만 세션 시작 시 자동 로드
  • ● 추가 내용은 주제별 파일에 저장, 필요 시 읽음
  • ● /memory 커맨드로 토글 및 관리 가능
메모리 관리 팁
  • ● /memory로 로드된 메모리 파일 확인 및 편집
  • ● 작업 완료 시마다 CLAUDE.md를 정기적으로 업데이트
  • ● 오래된 항목을 제거하여 혼란 방지
  • ● 세션 종료 전: "CLAUDE.md에 오늘 작업 내용과 중요 결정사항 업데이트해줘"

멀티파일 작업

실제 프로젝트의 파일 간 의존성 관리

Claude Code의 파일 이해 프로세스

Claude Code는 요청 시 다음 단계를 자동 수행한다:

  1. 디렉토리 구조 파악
  2. 관련 파일 식별
  3. import/require 관계 추적
  4. 변경 계획 수립

사용자가 모든 관련 파일을 명시할 필요는 없다.

효과적인 작업 요청 방법

기능 단위 요청

사용자 인증 기능을 추가해줘.
로그인, 회원가입, 로그아웃 포함.
기존 User 모델 활용해서.

범위 명시

src/auth/ 디렉토리 중심으로 JWT 구현해줘.
src/users/ 는 건드리지 마.

단계별 진행

한 번에 너무 많은 파일을 변경하지 않고, "1단계: 타입 정의만 먼저" 식으로 분할한다.

대규모 리팩토링 활용

Claude Code는 네이밍 변경, 패턴 일관성 적용, 의존성 교체 등의 대규모 리팩토링에서 강점을 발휘한다.
