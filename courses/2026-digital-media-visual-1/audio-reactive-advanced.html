TOP-CHOP 복습 & 오디오 리액티브 비주얼

중간평가에서 오디오 비주얼 작업을 수행하기 위한 준비 수업. TOP과 CHOP의 핵심 개념을 복습하고, 오디오 리액티브 비주얼의 전반적인 구조와 기법을 집중적으로 다룬다.

학습 목표
  • ● TOP과 CHOP의 핵심 개념을 오디오 리액티브 맥락에서 재정리할 수 있다
  • ● 오디오 신호의 특성(볼륨, 주파수, 비트)을 이해하고 각각을 시각 파라미터에 매핑할 수 있다
  • ● Audio Spectrum과 Audio Band를 활용한 주파수 기반 비주얼을 만들 수 있다
  • ● Feedback + Displace + 오디오를 결합한 복합적인 리액티브 비주얼을 설계할 수 있다
  • ● 중간평가 과제를 위한 오디오 비주얼 작업의 기본 구조를 갖출 수 있다

Part 01

TOP-CHOP 빠른 복습

오디오 리액티브 비주얼을 만들기 전에, 지금까지 배운 TOP과 CHOP의 핵심 노드를 빠르게 정리한다.

1-1. TOP 핵심 복습

핵심 개념
TOP(Texture Operator)은 2D 이미지/영상을 다루는 오퍼레이터다. 이미지를 입력받아 → 처리하고 → 이미지를 출력한다.
지금까지 배운 TOP 정리
분류 TOP 역할
소스Movie File In영상/이미지 파일을 불러온다
소스Noise TOP절차적 노이즈 패턴을 생성한다
소스Ramp TOP그라데이션 이미지를 생성한다
소스Constant TOP단색 이미지를 만든다
처리Transform이동, 회전, 스케일을 적용한다
처리Level밝기, 감마, 불투명도를 조절한다
처리Composite / Over여러 이미지를 합성한다
처리HSV Adjust색상, 채도, 명도를 조절한다
특수Feedback이전 프레임의 결과를 다시 가져온다 (잔상)
특수Displace이미지를 다른 이미지 기반으로 왜곡한다
제어Switch여러 입력 중 하나를 선택한다
정리Null네트워크 정리용 출력 노드

핵심 개념 ① Feedback Loop — 잔상과 누적의 원리

Feedback TOP은 네트워크 하류의 결과를 다시 상류로 가져와서, 이전 프레임 위에 현재 프레임을 덧쌓는 루프를 만든다. 이것이 잔상, 누적, 무한 터널 등 다양한 시각 효과의 기본 원리다.

Feedback 루프의 기본 구조

Feedback TOP → Composite (Over) → Level (opacity < 1) → Null (Target)
↑ ──────── 이전 프레임이 돌아옴 ──────── ↲
주의
Feedback 루프 안에서 Level TOP의 opacity를 반드시 1보다 낮게 설정해야 한다. 그렇지 않으면 이미지가 무한히 쌓여서 화면이 하얗게 된다. Transform을 추가해서 scale 0.98 + rotate 1°를 주면 잔상이 소용돌이치는 효과가 생긴다.

핵심 개념 ② Displacement — 이미지로 이미지를 왜곡하기

Displace TOP은 두 개의 입력을 받는다: Source(왜곡될 원본)와 Displace Image(왜곡의 패턴). Displace Image의 밝기 값이 Source의 픽셀을 밀어낸다. Noise TOP을 Displace Image로 쓰면 유기적인 물결 왜곡이 생기고, Displace Weight 파라미터로 왜곡 강도를 제어한다.

Movie File In → Displace TOP → Null
↑
Noise TOP (Displace Image)
오디오와 결합
오디오 리액티브에서 Displace Weight를 소리의 볼륨이나 주파수 값으로 제어하면, 소리가 클 때 화면이 일렁이고 조용할 때 안정되는 반응형 왜곡을 만들 수 있다.

핵심 개념 ③ Timemachine TOP — 시간 조작

Timemachine TOP은 입력 영상의 프레임을 Texture 3D TOP에 시간순으로 쌓아두고, 두 번째 입력의 밝기 값을 기준으로 원하는 시점의 프레임을 꺼내 보여주는 오퍼레이터다. 밝기 0(검정) = 가장 오래된 프레임, 밝기 1(흰색) = 가장 최근 프레임이다. 이 원리로 영상의 속도를 바꾸거나, 되감거나, 특정 순간에 멈추는 효과를 만든다.

Timemachine 네트워크 구조

Video Device In → Texture 3D TOP → Timemachine TOP → Null
↑
Ramp TOP (시간 위치 제어)
작동 원리
Texture 3D TOP은 입력 영상의 프레임을 3D 텍스처의 깊이(depth) 방향으로 쌓아둔다 — 시간의 기록장이다. Timemachine TOP은 첫 번째 입력으로 이 Texture 3D를 받고, 두 번째 입력(Ramp 등)의 밝기 값으로 어느 시점의 프레임을 보여줄지 결정한다. Ramp를 균일한 그라데이션으로 두면 현재가 그대로 나오고, Ramp 대신 오디오 값이나 노이즈를 넣으면 시간이 뒤틀리기 시작한다.
오디오와 결합
Ramp 대신 오디오 볼륨을 Constant TOP의 밝기로 변환해서 Timemachine의 두 번째 입력에 넣으면, 소리가 커질수록 과거 프레임을 보여주고 조용해지면 현재로 돌아오는 '시간 스크러빙' 효과를 만들 수 있다. Noise TOP을 넣으면 픽셀마다 다른 시점의 프레임이 섞여 글리치 같은 시간 왜곡 효과가 생긴다.
오디오 리액티브에서 자주 쓰는 TOP 조합
Feedback + Level + Composite — 잔상 효과
Feedback + Displace + Noise — 유기적 왜곡
Texture 3D + Timemachine + 오디오 — 시간 스크러빙
HSV Adjust + Level — 소리에 반응하는 색상/밝기 변화

1-2. CHOP 핵심 복습

핵심 개념
CHOP(Channel Operator)은 시간에 따라 변하는 숫자(채널)를 다루는 오퍼레이터다. 숫자를 생성하고, 가공하고, 다른 오퍼레이터의 파라미터에 연결하는 것이 핵심이다.
인터랙션의 기본 구조

입력 (센서/오디오) → 가공 (Math/Filter) → 출력 (Null) → TOP 파라미터
지금까지 배운 CHOP 정리
분류 CHOP 역할
입력Audio Device In마이크/사운드카드에서 오디오 입력을 받는다
입력Audio File In오디오 파일을 재생한다
생성LFO주기적으로 반복하는 값을 생성한다
생성Constant고정된 숫자 값을 만든다
분석Analyze최대, 최소, 평균, RMS 등을 계산한다
분석Audio Spectrum오디오의 주파수 스펙트럼을 분석한다
가공Math값의 범위를 변환한다 (Range, Gain 등)
가공Filter값을 부드럽게 만든다 (smoothing)
가공Select특정 채널만 선택한다
가공Null네트워크 정리 / 결과 출력

1-3. 복습 실습: 30초 만에 리액티브 구조 만들기

기본 리액티브 구조

LFO → Math → Null → ref → Transform / Level
LFO CHOP을 만들고 Frequency를 0.5로 설정한다
Math CHOP을 연결하고 Range 탭에서 출력 범위를 0~1로 맞춘다
Null CHOP을 연결한다
Transform TOP의 Scale 파라미터에 op('null1')['chan1'] 을 입력한다
확인 포인트
이미지가 LFO의 주기에 맞춰 부드럽게 커졌다 작아지면 성공! 이 생성 → 가공 → 출력 → Reference 구조가 모든 리액티브 비주얼의 기본이다.

오디오 리액티브 — 소리를 이해하기

2-1. 오디오 리액티브란?

2-2. Audio Device In CHOP — 소리 받기

Audio Device In CHOP은 마이크나 사운드카드에서 실시간 오디오 신호를 받아오는 입력 노드다. 출력은 -1 ~ 1 범위의 오디오 웨이브폼(파형)이며, 왼쪽/오른쪽 두 개의 채널을 가진다.

2-3. 볼륨 추출 — Analyze CHOP

RMS Power
Analyze CHOP의 Function을 RMS Power로 설정하면, 오디오 신호의 '평균적인 에너지'를 하나의 숫자(0~1)로 추출한다. 이것이 곧 볼륨이다. 소리가 크면 값이 높아지고, 조용하면 0에 가까워진다.
볼륨 추출 네트워크

Audio Device In → Analyze (RMS) → Math (Range) → Null
Audio Device In CHOP으로 오디오 입력을 받는다
Analyze CHOP을 연결하고 Function을 RMS Power로 설정
Math CHOP의 Range 탭에서 From Range를 0~0.5, To Range를 0~1로 설정 (소리에 맞게 조절)
Null CHOP을 연결하고, Level TOP의 Opacity에 op('null1')['chan1'] 입력
실습 포인트
소리가 클 때 밝아지고, 조용할 때 어두워지는지 확인한다. Math CHOP의 From Range를 조절해서 반응 감도를 맞춘다.

2-4. 주파수 분석 — audioAnalysis (Palette)

소리는 다양한 주파수가 섞여 있다. 저주파(Low)는 베이스/킥 드럼 같은 깊은 소리, 중주파(Mid)는 보컬/멜로디, 고주파(High)는 하이햇/심벌 같은 날카로운 소리다. 이것을 직접 분리하려면 Audio Spectrum → Select CHOP을 복잡하게 세팅해야 하지만, TouchDesigner의 Palette에서 제공하는 audioAnalysis 컴포넌트를 사용하면 간편하게 해결된다.

audioAnalysis 컴포넌트
Palette > Tools > audioAnalysis를 네트워크에 드래그하면 끝. 오디오 입력을 자동으로 분석하여 Low / Mid / High (주파수 대역별 에너지)와 Kick / Snare / Rhythm (비트 감지)을 바로 출력한다. 각 채널마다 Thresh, Gain, Smooth 노브로 감도를 조절할 수 있다.
Palette 브라우저를 열고 (Alt+L), Tools 폴더에서 audioAnalysis를 찾는다
네트워크 에디터에 드래그하여 배치한다
컴포넌트의 CHOP 출력에서 원하는 채널을 Select CHOP으로 골라 사용한다
audioAnalysis 출력 채널
채널 유형 설명
low연속값저주파 에너지 (베이스, 킥 드럼)
mid연속값중주파 에너지 (보컬, 멜로디)
high연속값고주파 에너지 (하이햇, 심벌)
kick이벤트킥 드럼 감지 (0 또는 1)
snare이벤트스네어 감지 (0 또는 1)
rhythm이벤트전체 리듬 감지
왜 audioAnalysis인가?
Audio Spectrum → Select CHOP으로 직접 주파수 대역을 자르려면 샘플 인덱스를 계산해야 하고, 적절한 범위를 찾기 어렵다. audioAnalysis는 이 과정을 자동으로 처리하고, 각 대역의 Gain과 Smooth까지 노브로 바로 조절할 수 있어서 훨씬 실용적이다.
주파수별 매핑 예시
주파수 대역 소리 예시 시각 매핑
저주파 (Low) 킥 드럼, 베이스 전체 스케일, 밝기 펄스
중주파 (Mid) 보컬, 멜로디 색상 변화, 회전 속도
고주파 (High) 하이햇, 심벌 노이즈 강도, 디테일 왜곡

2-5. 비트 감지 — Logic & Trigger CHOP

볼륨이나 주파수는 연속적인 값이지만, 비트는 순간적인 이벤트다. 볼륨 값이 특정 임계값(threshold)을 넘는 순간을 감지하여 '비트가 왔다!'는 신호를 만든다.

비트 감지 네트워크

Analyze (RMS) → Logic (Threshold) → Trigger → Null
Logic CHOP의 Convert를 Off When Outside Bounds로 설정, Bound Min을 0.3 정도로 설정
Trigger CHOP을 연결 — Attack, Sustain, Decay 값으로 반응 모양을 조절
비트 순간에 색 반전, 글리치 효과, 순간 전환 등에 활용
활용 예
Trigger CHOP의 출력을 Switch TOP의 Index에 연결하면, 비트에 맞춰 두 영상 사이를 순간 전환할 수 있다. 또는 HSV Adjust의 Hue Offset에 연결하면 비트마다 색이 바뀐다.
Part 03

오디오 리액티브 비주얼 실습

볼륨, 주파수, 비트를 활용한 실습을 단계별로 진행하고, 마지막에는 자신만의 오디오 리액티브 비주얼을 만든다.

3-1. 실습 1: 볼륨 리액티브 — 소리로 숨 쉬는 영상

Practice 1 볼륨 → 밝기 + 스케일 ▸
네트워크 구조

Transform TOP의 Scale에 op('null1')['chan1'] 입력

체크포인트

3-2. 실습 2: 주파수 리액티브 — 저음/고음 분리 반응

Select CHOP 두 개로 저주파 대역과 고주파 대역을 분리한다
각각 Analyze → Math → Null을 붙여 0~1 값으로 만든다
저주파 Null → Transform TOP의 Scale에 연결
고주파 Null → Displace TOP의 Weight에 연결
포인트
베이스가 울릴 때 영상이 펄스하고, 하이햇이 칠 때 영상이 왜곡되면 성공. 저음과 고음이 서로 다른 시각 요소를 움직이는 것이 핵심이다.

3-3. 실습 3: 비트 반응 + Feedback

Practice 3 비트 → Feedback 잔상 강도 ▸
네트워크 구조

Analyze (RMS) → Logic → Trigger → Null

Feedback → Composite → Level (← beat) → Displace → Null (Target)
비트 감지 체인을 만든다: Analyze → Logic → Trigger → Null
Feedback + Composite + Level + Displace + Null 구조를 만든다
비트 Null을 Level TOP의 Opacity에 Reference로 연결
Trigger의 Attack을 짧게 (0.05), Decay를 길게 (0.5) 설정하여 비트에 맞춘 잔상 효과를 만든다
포인트
비트가 올 때 잔상이 강해지고, 빠지면 서서히 사라지는 '호흡' 같은 느낌이 핵심이다. Displace Weight도 비트에 연결하면 더 역동적인 효과가 나온다.

오디오 리액티브 작업을 위한 팁

오디오 리액티브 비주얼을 더 완성도 있게 만들기 위한 실전 노하우.

4-1. 값 다듬기가 전부다

오디오 리액티브에서 가장 중요한 것은 '어떤 노드를 쓰느냐'가 아니라, 추출한 값을 얼마나 잘 다듬느냐이다. 아래는 흔한 문제와 해결 방법이다.

트러블슈팅
문제 원인 해결
반응이 너무 약하다 값 범위가 너무 좁다 Math CHOP의 From Range를 줄인다 (예: 0~0.1)
반응이 너무 과하다 값 범위가 너무 넓다 Math CHOP의 To Range를 좁히거나 Clamp 활성화
움직임이 떨린다 오디오 신호의 노이즈 Filter CHOP의 Width를 높인다 (0.3~0.5)
반응이 너무 느리다 필터가 너무 강하다 Filter Width를 낮춘다 (0.05~0.1)
조용할 때도 반응한다 노이즈 플로어 Math CHOP의 From Range 하한을 올린다 (예: 0.02~)

4-2. 소리마다 다른 필터 속도

핵심 원칙
모든 오디오 속성에 같은 Filter Width를 쓰면 안 된다. 소리의 성격에 맞는 필터 속도를 각각 다르게 설정해야 자연스러운 반응이 나온다.
권장 Filter Width
소리 속성 Filter Width 이유
비트 / 킥 드럼 0.02 ~ 0.08 빠르게 반응해야 펀치감이 있다
볼륨 / 전체 에너지 0.1 ~ 0.3 중간 정도의 부드러움이 자연스럽다
분위기 / 앰비언스 0.5 ~ 1.0 느리게 변해야 배경 같은 느낌이 난다

4-3. 음악 선택도 작업의 일부

중요
오디오 리액티브 비주얼의 품질은 음악 선택에 크게 좌우된다. 리듬이 명확하고, 저음과 고음이 잘 분리된 음악이 오디오 리액티브에 적합하다. 주파수 대역이 뚜렷하지 않은 앰비언트/드론 음악은 볼륨 기반 반응에 더 적합하다.
