CHOP / Audio Reactive 기초

TOP 복습 위에 "시간에 따라 변하는 숫자"라는 CHOP의 핵심 개념을 이해하고, 오디오 입력을 분석하여 소리에 반응하는 비주얼을 직접 만들어본다.

  • ● Feedback TOP과 Displace TOP의 원리를 이해하고 활용할 수 있다
  • ● CHOP이 무엇인지 이해하고, TOP과의 차이를 설명할 수 있다
  • ● CHOP Reference의 세 가지 방식(Export, Reference, Bind)을 이해한다
  • ● 입력 → 가공 → 출력 이라는 인터랙션의 기본 구조를 체득한다
  • ● Math CHOP을 활용해 값의 범위를 변환할 수 있다
  • ● 오디오 입력으로 Feedback/Displace를 포함한 영상 파라미터를 실시간 제어할 수 있다

Part 01

TOP 복습 — Feedback과 Displacement

이전 수업에서 배운 TOP의 핵심 개념을 복습하고, 두 가지 강력한 이미지 처리 도구를 추가로 익힌다.

지난 수업 복습: TOP의 기본 개념

핵심 개념
TOP(Texture Operator)은 TouchDesigner에서 2D 이미지/영상을 다루는 오퍼레이터 패밀리다. 모든 TOP은 이미지를 입력받아 → 처리하고 → 이미지를 출력한다. 와이어로 연결하면 이미지가 흘러간다.

Ramp그라데이션(경사) 이미지를 생성한다

Feedback TOP — 이미지의 잔상과 누적

Feedback TOP은 네트워크의 하류(downstream)에 있는 TOP의 결과를 다시 상류(upstream)로 가져오는 특별한 오퍼레이터다. 일반적으로 TouchDesigner는 순환(loop) 연결을 허용하지 않지만, Feedback TOP은 이것을 안전하게 가능하게 한다. 쉽게 말하면: 이전 프레임의 결과물 위에 현재 프레임을 덧쌓는 것이다.

Feedback 루프 원리

Feedback TOP ──→ 처리 체인 ──→ Target TOP
↑ ─────────── 이전 프레임 ─────────── ↲
시각적 비유
TV 화면 앞에 카메라를 놓는 것과 같다. 카메라가 화면을 찍고 → 그 영상이 화면에 나오고 → 다시 카메라가 찍고... 반복될수록 이미지가 축소되고, 회전하고, 색이 변하면서 무한 터널 같은 효과가 나온다.
핵심 파라미터
파라미터설명
Target TOP피드백 루프의 끝점. 이 TOP의 출력이 다음 프레임에 Feedback TOP의 출력이 된다
Reset0이면 피드백 활성, 1이면 비활성(입력 이미지를 그대로 통과)
Reset Pulse클릭하면 한 프레임 동안 피드백을 리셋한다

Displace TOP — 이미지를 왜곡하는 힘

Displace TOP은 하나의 이미지를 다른 이미지로 왜곡(warp)하는 오퍼레이터다. 두 개의 입력을 받는다: Source Image(왜곡될 원본)와 Displace Image(왜곡의 방향과 세기를 결정).

원리
Displace Image의 밝기 값이 Source Image의 픽셀을 이동시킨다. 50% 회색(0.5) = 이동 없음(기준점), 밝은 부분 = 한 방향으로 이동, 어두운 부분 = 반대 방향으로 이동. Red 채널이 X축, Green 채널이 Y축 방향 이동을 결정한다.
시각적 비유
물 위에 비친 풍경을 상상해보자 — 물결이 없으면 풍경이 그대로 보이고(Weight = 0), 잔잔한 물결이면 살짝 일렁이고(작은 값), 거센 파도면 심하게 뒤틀린다(큰 값). 물결의 패턴(Displace Image)이 왜곡의 모양을 결정한다.

기본 사용법: Noise로 왜곡하기

Movie File In → Displace TOP → Null
↑
Noise TOP (Displace Image)
Noise TOP 설정에 따른 왜곡 변화
Noise TOP 파라미터효과
Period 크게넓고 부드러운 왜곡 (물 위의 물결)
Period 작게좁고 날카로운 왜곡 (유리 깨짐)
Translate 변화왜곡 패턴이 움직인다 (흐르는 물)
Amplitude밝기 차이 → 왜곡 강도
TypePerlin, Simplex 등 — 왜곡의 질감이 달라진다
팁
Noise TOP의 Translate 파라미터에 absTime.seconds * 0.1을 넣으면 시간에 따라 노이즈가 움직인다.

Feedback + Displace: 결합의 힘

Feedback과 Displace를 결합하면 단순한 잔상이나 왜곡을 넘어서, 살아 움직이는 듯한 유기적 비주얼을 만들 수 있다. 매 프레임마다 이전 결과가 살짝 왜곡되면서 다시 쌓인다 → 시간이 지나면서 이미지가 흘러가고, 녹아내리고, 퍼져나가는 효과.

Feedback + Displace 기본 구조

Feedback → Level → Displace → Composite → Null
opacity: 0.95 ↑ ↑ 원본 이미지
Noise
파라미터 감각
설정낮은 값높은 값

Displace Weight살짝 일렁임 (0.005)심하게 뒤틀림 (0.1)
Noise Period날카로운 왜곡부드러운 왜곡

미리보기
이 값들을 나중에 CHOP으로 제어하면 소리에 반응하는 비주얼이 된다. 예: 소리가 크면 Displace Weight가 올라가고, 조용하면 내려간다.
Part 02

CHOP — 시간에 따라 변하는 숫자

CHOP(Channel Operator)의 핵심 개념과 세 가지 역할(생성, 가공, 출력)을 이해한다.

CHOP이란 무엇인가

TOP

다루는 것: 2D 이미지 (픽셀)

시각적 표현: 이미지 미리보기

와이어 색상: 보라색

핵심 질문: "이 이미지를 어떻게 바꿀까?"

CHOP

다루는 것: 숫자의 흐름 (채널)

시각적 표현: 그래프 (파형)

와이어 색상: 녹색

핵심 질문: "이 숫자를 어떻게 만들까?"

핵심 정의
CHOP = Channel Operator = 시간에 따라 변하는 숫자를 만들고, 가공하고, 전달하는 도구. TOP이 사진이라면, CHOP은 심전도 그래프다.
Channel (채널)

하나의 데이터 흐름. 마우스 x좌표 = 1개 채널, 오디오 L/R = 2개 채널.

Sample (샘플)

매 순간 기록되는 하나의 값. 그래프의 한 점 = 하나의 샘플.

Sample Rate

1초에 몇 번 값을 기록하는가. 컨트롤: 60Hz, 오디오: 44100Hz.

CHOP Viewer 습관
CHOP을 처음 배울 때 가장 중요한 습관: 항상 Viewer를 열어두고 값이 어떻게 움직이는지 관찰한다. 가로축 = 시간, 세로축 = 값, 각 선 = 하나의 채널.

CHOP의 세 가지 역할: 생성, 가공, 출력

생성 Generator → 가공 Filter → 출력 Output/Export
CHOP 오퍼레이터 분류
역할하는 일대표 CHOP
생성값을 만들어낸다LFO, Noise, Constant, Pattern
입력외부 데이터를 가져온다Mouse In, Keyboard In, Audio Device In
가공값을 변환/처리한다Math, Filter, Lag, Limit, Logic, Analyze
출력가공된 값을 전달한다Null (+ Export/Reference)
인터랙션의 기본 구조
모든 인터랙티브 작업은 하나의 구조로 요약된다:
입력(Input) → 변환(Process) → 출력(Output)

이 구조를 몸에 익히면 나중에 입력 장치가 바뀌어도(마우스 → 센서 → 카메라 → MIDI) 같은 방식으로 응용할 수 있다.

생성 계열 CHOP — 값을 만든다

LFO CHOP

Low Frequency Oscillator = 일정한 주기로 반복되는 값을 만든다.

Frequency1초에 몇 번 반복 (Hz)
Amplitude파형의 크기
Offset파형의 중심값 이동
TypeSin, Square, Triangle, Ramp
Noise CHOP

유기적이고 불규칙한 변화. LFO가 "기계적 반복"이라면 Noise는 "자연스러운 흔들림".

Period노이즈의 주기. 클수록 변화가 느리다
Amplitude출력값의 범위
Seed같은 설정이라도 다른 패턴
파형 종류와 느낌

Sin (사인파):   ～～～～～ → 부드럽고 자연스러운. 호흡, 물결
Square (사각파): ▔▁▔▁▔▁ → 깜빡임, ON/OFF, 스트로보
Triangle (삼각파): /\/\/\/\ → 일정한 속도로 왕복. 기계적
Ramp (톱니파):  /|/|/|/| → 한 방향으로 올라갔다 순간 돌아옴

입력 계열 CHOP — 외부 데이터를 가져온다

Mouse In

tx ty lselect mselect rselect
별도 장비 없이 바로 실습 가능한 가장 기본적인 입력.

Audio Device In

마이크나 시스템 오디오를 가져온다. 오디오 리액티브의 시작점.
Mac: BlackHole 등 가상 드라이버 필요할 수 있음

Audio Spectrum

FFT 기반 주파수 대역별 분리. 저음(bass), 중음(mid), 고음(high)을 각각 다른 파라미터에 연결.

FFT란
Fast Fourier Transform — 복잡한 파형을 주파수별 성분으로 분해하는 수학적 방법. 합주를 들으면서 "베이스 기타가 얼마나 크게 나오는가", "심벌즈가 얼마나 크게 나오는가"를 각각 추출하는 것과 같다.

가공 계열 CHOP — 값을 다듬는다

Math CHOP ⭐ 가장 중요

범위 변환, 반전, 오프셋, 배율 조정 — 거의 모든 인터랙션에서 필요하다.

핵심 인사이트
인터랙션이 안 되는 대부분의 경우는 값을 못 받는 게 아니라, 받은 값을 적절한 범위로 못 바꾸는 것이다.
입력실제 범위필요한 범위
마우스 X0 ~ 1920-1 ~ 1 (Transform tx)
오디오 RMS0 ~ 0.010 ~ 2 (Level brightness)
센서값300 ~ 7000 ~ 1 (opacity)

Range 탭: From Range(입력값 범위) → To Range(출력값 범위). 이것만 알면 대부분의 범위 변환 문제를 해결할 수 있다.

Analyze CHOP

복잡한 신호를 하나의 제어값으로 요약한다.

RMS Power(에너지), Maximum(피크), Average(평균). 오디오 리액티브에서 가장 많이 쓰는 건 RMS Power.

Lag CHOP

거친 값을 부드럽게 만든다 (스무딩). 

스무딩 전: ╱╲╱╲╱╲╱╲  (들쭉날쭉)
스무딩 후: ～～～～～  (부드러움)
Logic CHOP

임계값(threshold) 기준으로 ON/OFF를 만든다. 

입력: 0.2 0.5 0.8 0.3 0.9
임계:     ── 0.6 ──
출력: 0   0   1   0   1
Trigger / Count

Trigger: 신호가 임계값을 넘는 순간 한 번만 반응(펄스).
Count: 입력 신호가 올라갈 때마다 카운트 증가 → Switch의 인덱스로 쓰면 컷 전환.

연속값과 사건(Event)의 구분

연속값 (Continuous)

매 순간 변하는 값. 부드럽게 변화한다.

마우스 위치, 센서 거리, 오디오 레벨

→ 밝기, 위치, 크기, 색상 제어에 적합

사건 (Event / Trigger)

특정 순간에 발생하는 신호. 0 또는 1.

키를 눌렀다, 임계값을 넘었다, 장면을 바꾼다

→ Switch, 재생 시작, 이펙트 발동에 적합

핵심
"입력값이 있다"와 "사건이 발생했다"는 다른 문제다. 이 구분이 명확해야 인터랙션 설계가 깔끔해진다.
Part 03

CHOP Reference — 값을 파라미터에 연결하는 법

CHOP에서 만든 값을 TOP의 파라미터에 연결하는 세 가지 방법을 배운다.

1. Export (드래그)

가장 직관적이고 빠른 방법.

CHOP 노드 우측 하단의 +를 드래그해서 파라미터 위에 놓는다.

녹색 배경 = Export 활성

수식 불가. 파라미터가 CHOP에 의해 잠긴다.

2. Reference (표현식)

파라미터 필드에 Python 표현식을 직접 입력.

op('chop이름')['채널이름']
파란색 배경 = Reference 활성

수식 가능! 가장 유연하다.

3. Bind (양방향)

CHOP과 파라미터를 양방향으로 연결.

파라미터를 바꾸면 CHOP도 바뀐다.

녹색 배경 (양방향)

고급 기능. 이번 수업에서는 소개만.

Reference 표현식 예시

표현식의미
op('lfo1')['chan1']lfo1의 chan1 채널 값
op('mouse1')['tx']마우스 X 위치
op('analyze1')['chan1']오디오 분석값
op('lfo1')['chan1'] * 2값을 2배로
op('audio1')['chan1'] * 10 + 110배 하고 1 더하기
언제 뭘 쓸까
처음 배울 때 → Export (드래그)
간단한 계산이 필요하면 → Reference (표현식)
양방향 제어가 필요하면 → Bind
Part 04

오디오 리액티브 비주얼

소리 데이터를 실시간으로 분석하여 영상 파라미터를 제어하는 오디오 리액티브 비주얼의 기초.

오디오 리액티브란

소리가 크면 밝아지고, 베이스가 강하면 화면이 흔들리고, 킥 드럼이 울릴 때마다 장면이 전환되는 — 이런 비주얼이 오디오 리액티브다.

오디오 분석의 기초 — 5단계 파이프라인

오디오 리액티브 기본 뼈대

Audio Device In → Analyze (RMS) → Math (범위변환) → Lag (스무딩) → Null → Export
1
오디오 입력받기

Audio Device In CHOP — Device에서 마이크/시스템 오디오 선택. Viewer에서 파형 확인.

2
에너지 추출하기

Analyze CHOP (RMS Power) — 빠르게 진동하는 오디오 파형 → 하나의 숫자(현재 볼륨)

3
범위 맞추기

Math CHOP (Range) — RMS 출력(0~0.01)을 원하는 범위(0~2)로 변환

4
부드럽게 만들기

Lag CHOP — Lag Up: 0.05, Lag Down: 0.3 → "빠르게 반응하고 천천히 사라지는" 느낌

5
파라미터에 연결하기

Null CHOP → Export — 이제 소리가 크면 밝아지고, 소리가 작으면 어두워진다.

인터랙션의 미감 — "어떻게 반응하는가?"

핵심 원칙
인터랙티브 작업에서는 기술적으로 "작동하는 것"과 미적으로 "좋은 것"이 다르다.

나쁜 예: 소리 → 즉시 밝기 100% → 거칠고 불쾌하다
좋은 예: 소리 → 0.1초에 밝기 상승 → 0.5초에 천천히 하강 → 호흡감이 있다
Lag CHOP 비대칭 설정
설정효과
Up: 0.05, Down: 0.3빠르게 반응하고 천천히 사라짐 → 펀치감 + 여운
Up: 0.3, Down: 0.05천천히 반응하고 빠르게 사라짐 → 몽환적
Up: 0.1, Down: 0.1대칭적 반응 → 중립적
인사이트
인터랙션은 "편집과 리듬의 문제"이기도 하다. 값 하나로 무엇을 얼마나 바꿀 것인가, 언제 반응하고 언제 하지 않을 것인가 — 이 선택이 영상의 인상과 감정을 결정한다.
Part 05

실습 — 오디오 리액티브 비주얼 만들기

단계별 실습으로 소리에 반응하는 비주얼을 직접 만든다.

실습 A

Feedback + Displace 체험

A-1 기본 잔상 만들기 ▸

목표: Feedback TOP의 동작 원리를 체감한다

Movie File In TOP으로 영상을 불러온다
Feedback TOP을 만든다
Composite TOP (Operation: Over)를 만들고: 입력 1에 Feedback, 입력 2에 원본 영상
Level TOP을 연결하고 opacity를 0.95로 설정
Null TOP을 끝에 연결
Feedback TOP의 Target TOP을 Level TOP으로 설정
확인 포인트
opacity를 0.8 / 0.95 / 0.99로 바꿔보면서 차이를 느껴본다. Reset Pulse로 초기화해본다.
A-2 Noise Displacement ▸

목표: Displace TOP으로 이미지를 왜곡하고, 패턴의 변화를 관찰한다

영상을 Displace TOP의 첫 번째 입력에 연결
Noise TOP을 두 번째 입력에 연결
Displace Weight를 0.02에서 시작
Noise TOP의 Transform → Translate X에 absTime.seconds * 0.1 입력
확인 포인트
Displace Weight를 0.01 / 0.05 / 0.2로 비교. Noise의 Period, Type을 바꿔보면서 질감 차이를 관찰.
A-3 Feedback + Displace 결합 ▸

목표: 두 효과를 결합하여 유기적으로 움직이는 비주얼을 만든다

실습 A-1의 Feedback 구조를 만든다
Level TOP과 Composite TOP 사이에 Displace TOP을 삽입
Displace의 두 번째 입력에 Noise TOP 연결
Displace Weight: 0.01, Level opacity: 0.96으로 시작
여러 파라미터를 바꿔가며 "마음에 드는 움직임"을 찾아본다
실습 B

CHOP 기본과 Reference 연습

B-1 LFO로 영상 자동 움직이기 ▸
LFO CHOP → Math CHOP → Export → Transform TOP (tx)
LFO CHOP을 만든다 (Type: Sin, Frequency: 0.3)
Math CHOP Range: From -1~1, To -0.5~0.5
Math CHOP의 +를 드래그해서 Transform TOP의 tx에 Export
B-2 Reference 표현식 연습 ▸
Level TOP의 Opacity 파라미터를 우클릭 → Expression Mode
다음 표현식을 입력한다:
op('math1')['chan1'] * 0.5 + 0.5
확인
Export(녹색)와 Reference(파란색)의 파라미터 색 차이를 확인한다.
B-3 마우스로 영상 제어 + 스무딩 ▸
Mouse In → Select (tx) → Math → Lag → Export → Transform (tx)
Math CHOP Range: From 0~1920, To -1~1
Lag CHOP 연결, Lag: 0.2로 설정
Transform TOP의 tx에 Export
확인
Lag 값을 0.05 / 0.2 / 1.0으로 바꿔보면서 차이를 느낀다. Lag가 없을 때 vs 있을 때를 비교.
실습 C–F

오디오 리액티브 비주얼

C 소리에 반응하는 밝기 ▸
Audio Device In → Analyze (RMS) → Math → Lag → Null → Export → Level (brightness)
Audio Device In → Device에서 마이크/시스템 오디오 선택
Analyze CHOP: Function을 RMS Power로
Math CHOP Range: From 0~0.01, To 0~2
Lag: Up 0.05, Down 0.3
Null CHOP 이름을 audio_level로 변경
Level TOP의 Brightness에 Export
D 소리에 반응하는 왜곡 (Displace) ▸

실습 C의 CHOP 네트워크를 그대로 사용. Displace TOP의 Weight에 Reference 표현식으로 연결한다. 

op('audio_level')['chan1'] * 0.05
확인
* 0.05 부분을 * 0.1, * 0.2로 바꿔보면서 감도를 조절한다.
E 소리에 반응하는 Feedback ▸

Feedback 루프 안의 파라미터에 오디오를 연결한다.

Level TOP Opacity에 Reference:

0.9 + op('audio_level')['chan1'] * 0.08

Displace Weight에 Reference:

op('audio_level')['chan1'] * 0.03

소리가 클 때 잔상이 더 오래 남고(opacity↑), 동시에 왜곡이 강해진다(weight↑). 조용하면 거의 원본만 보인다.

F 임계값 기반 장면 전환 ▸
연속값 + 사건 복합 구조

audio_level → [연속값] → Level / Displace

audio_level → Logic → Count → Switch TOP (index)
Logic CHOP: Off When 0.4, On When 0.6
Count CHOP: Limit Loop, Min 0, Max 영상 개수-1
Count → Switch TOP의 Index에 Export
실습 G

미니 과제 — 나만의 오디오 리액티브 비주얼

과제 조건
  • ● 오디오 입력 사용 필수
  • ● 가공 CHOP 2개 이상 사용 (Math, Lag, Limit, Logic 등)
  • ● TOP 파라미터 2개 이상 제어
  • ● 연속값 제어 1개와 이벤트 제어 1개를 모두 포함
  • ● Feedback TOP 또는 Displace TOP 중 1개 이상 활용
아이디어 예시
아이디어연속값이벤트
오디오 + Feedback 잔상소리로 opacity/왜곡킥에 반응해 색상 전환
오디오 + Displacement소리로 Weight 조절큰 소리에 글리치 전환
마우스 + 오디오 복합마우스→위치, 오디오→밝기키보드로 영상 전환
주파수 분리저음→스케일, 고음→색상전체 볼륨 임계값→전환
Part 06

정리와 다음 단계

핵심 원칙

1. 인터랙션은 "값을 설계하는 것"이다

영상을 직접 건드리는 것보다 영상을 움직이는 값을 설계하는 것이 인터랙션이다.

2. 좋은 인터랙션은 가공 방식에서 나온다

마우스를 받는 것은 쉽다. 그 값을 적당한 속도, 범위, 감도, 리듬으로 가공하는 것이 어렵고 중요하다.

3. 센서 데이터는 대부분 더럽고 불안정하다

"센서가 이상하다"라고 할 때 사실은 데이터 가공 문제인 경우가 많다.

4. 인터랙션은 서사와 편집의 문제이기도 하다

값 하나로 무엇을 얼마나 바꿀 것인가, 언제 반응하고 언제 하지 않을 것인가 — 이 선택이 영상의 인상과 감정을 결정한다.
