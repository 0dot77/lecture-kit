Overview

수업 개요

이전 수업에서 배운 TOP의 핵심 개념을 복습하고, Feedback TOP과 Displace TOP이라는 두 가지 강력한 이미지 처리 도구를 복습한다.

그 위에 "시간에 따라 변하는 숫자"라는 CHOP의 핵심 개념을 이해하고, CHOP Reference를 통해 값을 영상 파라미터에 연결하는 방법을 배운다.

최종적으로 오디오 입력을 분석하여 Feedback과 Displacement 효과가 소리에 반응하는 비주얼을 직접 만들어본다.

Feedback TOP

이미지의 잔상과 누적

Feedback TOP은 네트워크의 하류(downstream)에 있는 TOP의 결과를 다시 상류(upstream)로 가져오는 특별한 오퍼레이터다.

이전 프레임의 결과물 위에 현재 프레임을 덧쌓는 것 — TV 화면 앞에 카메라를 놓는 것과 같다.

파라미터설명
Target TOP피드백 루프의 끝점. 이 TOP의 출력이 다음 프레임에 Feedback TOP의 출력이 된다
Reset0이면 피드백 활성, 1이면 비활성
Reset Pulse클릭하면 한 프레임 동안 피드백을 리셋
Feedback TOP —→ [처리 체인] —→ Target TOP ↑ | ←———— 이전 프레임 ————”
Feedback — 작동 원리

Feedback TOP 동작 단계

  1. Feedback TOP은 처음에 입력 이미지(Input)를 통과시킨다
  2. 활성화되면, Target TOP으로 지정된 하류 노드의 출력을 자신의 출력으로 사용한다
  3. 결과적으로 이전 프레임의 처리 결과가 다음 프레임의 입력이 된다
  4. 매 프레임 반복 → 잔상, 트레일, 누적 효과
Feedback — Example

잔상 만들기

Movie File In → Feedback → Composite (Over) → Level (0.95) → Null

단계별 설명

  1. Feedback TOP을 만들고, Target TOP을 하류의 Level TOP으로 지정
  2. Composite TOP의 입력 1: Feedback, 입력 2: 원본 영상
  3. Level TOP의 opacity를 0.95 정도로 낮춤
  4. 결과: 이전 프레임이 살짝 투명해지면서 새 프레임이 위에 쌓인다 → 잔상 효과

주의

Level TOP의 opacity를 반드시 1보다 낮게 설정해야 한다. 그렇지 않으면 이미지가 계속 쌓여서 화면이 하얗게 된다.

Displace TOP

이미지를 왜곡하는 힘

하나의 이미지를 다른 이미지로 왜곡(warp)하는 오퍼레이터. 두 개의 입력:

입력역할
Source Image (1st)왜곡될 원본 이미지
Displace Image (2nd)왜곡의 방향과 세기를 결정하는 이미지

시각적 비유 — 물 위의 풍경: 물결의 패턴(Displace Image)이 왜곡의 모양을 결정한다. Displace Image의 밝기 값이 Source 픽셀을 이동시킨다.

Movie File In → Displace → Null
↑ Noise TOP 연결

50% 회색 (0.5) → 이동 없음 (기준점)
밝은 부분 → 한 방향으로 이동
어두운 부분 → 반대 방향으로 이동

Displace — Example

Noise로 왜곡하기

Movie File In → Displace → Null
↑ Noise TOP (Displace Image)
Noise TOP 파라미터효과
Period 크게넓고 부드러운 왜곡 (물 위의 물결)
Period 작게좁고 날카로운 왜곡 (유리 깨짐)
Translate 변화왜곡 패턴이 움직인다 (흐르는 물)
Type 변경Perlin, Simplex, Sparse → 왜곡의 질감이 달라진다

Noise TOP의 Translate X에 absTime.seconds * 0.1을 넣으면 시간에 따라 노이즈가 움직인다.

Displace Weight를 0.01~0.05에서 시작. 너무 크면 이미지가 완전히 깨진다.

CHOP Fundamentals

CHOP의 세 가지 핵심 요소

Channel (채널)

하나의 데이터 흐름

마우스 x좌표, 오디오 볼륨

Sample (샘플)

매 순간 기록되는 하나의 값

현재 프레임에서의 마우스 x

Sample Rate

1초에 몇 번 값을 기록하는가

컨트롤: 60Hz / 오디오: 44100Hz

CHOP을 처음 배울 때 가장 중요한 습관: 항상 Viewer를 열어두고 값이 어떻게 움직이는지 관찰한다.

CHOP Categories

CHOP의 세 가지 역할

생성 Generator → 가공 Filter → 출력 Output
역할하는 일대표 CHOP

Generator CHOP

LFO CHOP — 반복하는 파형

LFO = Low Frequency Oscillator. 일정한 주기로 반복되는 값을 만든다.

파라미터설명
Frequency1초에 몇 번 반복 (Hz). 0.5 = 2초에 한 번
Amplitude파형의 크기. 1이면 -1~1 범위
TypeSin / Square / Triangle / Ramp

Frequency를 아주 낮게 (0.05~0.2) 설정하면 천천히 숨 쉬는 듯한 움직임이 나온다.

Generator CHOP

Noise CHOP — 유기적 변화

유기적이고 불규칙한 변화를 만든다. LFO가 "기계적 반복"이라면 Noise는 "자연스러운 흔들림"이다.

LFO

  • 패턴이 정확히 반복

  • 펄스, 호흡, 회전

Noise

  • 반복하지 않음

Period클수록 변화가 느리다
Amplitude출력값의 범위
Seed같은 설정이라도 Seed가 다르면 다른 패턴

Continuous vs Event

연속값 vs 사건(Event)

연속값

매 순간 변하는 값. 부드럽게 변화한다.

값에 따라 부드럽게 변화

사건 (Event)

특정 순간에 발생하는 신호. 0 또는 1.

임계값을 넘으면 ON/OFF

"입력값이 있다"와 "사건이 발생했다"는 다른 문제다. 이 구분이 명확해야 인터랙션 설계가 깔끔해진다.

CHOP에서 만든 숫자를 TOP의 파라미터로 보내는 세 가지 방법

CHOP Reference

CHOP Reference란

CHOP에서 만들고 가공한 값을 TOP(또는 다른 오퍼레이터)의 파라미터에 연결하는 것.

값을 만들고 (생성/입력) → 값을 다듬고 (가공) → 값을 연결한다 (Reference/Export)
방법난이도방향특징
Export (드래그)쉬움CHOP → 파라미터가장 간단. 파라미터가 잠긴다
Reference (표현식)중간CHOP → 파라미터수식 가능. 유연하다
Bind어려움양방향파라미터를 바꾸면 CHOP도 바뀐다
Method 1 — 가장 쉽다

드래그 익스포트 (Export)

사용법

  1. CHOP 노드의 우측 하단에 마우스를 올리면 + 아이콘이 나타난다
  2. 이 +를 클릭하고 드래그해서 연결하고 싶은 파라미터 위에 놓는다
  3. 드롭하면 채널 선택 메뉴가 나온다
  4. 원하는 채널을 선택하면 연결 완료

연결된 파라미터는 녹색 배경으로 표시. CHOP이 제어 중이므로 수동 조작 불가.

해제: 파라미터를 우클릭 → Unexport

Part 6

Summary
정리와 다음 단계

Principles

핵심 원칙 4가지

1. 인터랙션은 "값을 설계하는 일"이다

TOP 수업이 화면을 만드는 수업이었다면, CHOP 수업은 화면을 움직이게 만드는 수업이다.

2. 좋은 인터랙션은 가공 방식에서 나온다

마우스를 받는 것은 쉽다. 그 값을 적당한 속도, 범위, 감도, 리듬으로 가공하는 것이 어렵고 중요하다.

3. 센서 데이터는 대부분 더럽다

실제 입력은 예쁘게 들어오지 않는다. 스무딩, 클램핑, 임계값 조절은 거의 필수.

4. 인터랙션은 서사와 편집의 문제

빠르게 튈 것인가, 천천히 따라올 것인가 — 이 선택이 영상의 인상과 감정을 결정한다.
