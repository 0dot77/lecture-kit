02 / processing

데이터를 정리하고 조건으로 바꾼다

Raw input은 그대로 쓸 수 없습니다. 노이즈를 줄이고, 스케일을 맞추고, 의미 단위로 묶어야 비로소 '제어 가능한' 값이 됩니다.

    스케일링 & 정규화

    0–1 범위로 맞추기. Math CHOP, Limit CHOP 활용.

  • 02

    스무딩 & 필터

    떨림 제거. Filter CHOP, Lag CHOP로 자연스러운 곡선 만들기.

  • 03

    조건 / 상태 변환

    임계값 통과 → 상태 변경. Logic CHOP + Python script.

  • 04

    스케줄링

    시간 기반 큐. Timer CHOP, Animation COMP로 장면 전환 타이밍 제어.

Part 01 · Concept 17 / 55
Interactive ≠ effects

인터랙티브는
결국 데이터
연결의 시스템

멋진 비주얼이 자동으로 인터랙티브한 무대를 만들지 않습니다.

입력 → 처리 → 출력이 의도된 방식으로 연결되어 있고, 그 연결이 공연 중 일관되게 작동할 때 비로소 인터랙티브합니다.

TouchDesigner는 그 연결 구조 자체를 실시간으로 만들고 조정하기 좋은 도구입니다.

Part 01 · Why TD 18 / 55
Why TouchDesigner

Real-time connection.

TD가 무대에서 자주 선택되는 이유는 "예쁜 비주얼"보다 연결 구조를 실시간으로 다룰 수 있는 워크플로우에 있습니다.

Node-based

시각적 데이터 흐름

노드 와이어로 흐름이 눈에 보임 → 디버깅과 협업이 쉬움.

Real-time

실시간 편집

멈추지 않고 수정 가능. 공연 직전 튜닝이 가능한 환경.

All-in-one

I/O 다 됨

비주얼·오디오·DMX·OSC·NDI·Serial 한 곳에서 처리.

Extensible

Python 내장

조건 / 상태 / 외부 API 호출 등 일반 코드도 자연스럽게 결합.

Part 03 · CHOP Ref 32 / 55
Connecting values

CHOP
Reference

UI에서 만든 값을 다른 파라미터로 보내는 가장 핵심 기법. "이 슬라이더 값을 저 노드의 파라미터에 묶기"입니다.

How
  1. 슬라이더 / 버튼 COMP의 출력 채널을 확인
  2. 연결할 파라미터에서 오른클릭 → Reference
  3. op('slider1')['v1'] 형식의 표현식 자동 생성
  4. 실시간으로 값이 따라옴
Use cases
  • 슬라이더 → Feedback TOP의 amount
  • 버튼 → Switch TOP의 index (장면 전환)
  • 오디오 RMS → 텍스트 크기

Part 05 · TD vs Web 46 / 55
Two kinds of UI

내부 UI vs Web UI

TouchDesigner UI

Container COMP 기반

  • + TD 내부 데이터에 즉시 접근
  • + 같은 머신에서 동작 → 지연 ↓
  • − 디자인 자유도 제한
  • − 다른 기기에서 못 봄
Web UI

HTML / CSS / JS

  • + 폰/태블릿/원격에서 접근
  • + 시각 디자인 자유도 ↑
  • + 오퍼레이터 친화적
  • − 네트워크 연결 필요

실제로는 둘을 섞어 씁니다. 내부 UI는 모니터링, Web UI는 오퍼레이션.

Part 05 · Cautions 49 / 55
Stage = strict environment

공연 시스템에서 AI를 쓸 때

무대는 실험실이 아닙니다. AI를 가운데에 두기 전에 다음 4가지를 통과해야 합니다.

01

실시간성

한 프레임이라도 늦으면 무대가 끊깁니다. AI 호출은 빠르지 않습니다.

02

안정성

실패 시 폴백이 있어야 합니다. AI는 가끔 무너집니다.

03

재현 가능성

같은 입력에 같은 출력이 필요. AI는 같은 입력에도 다르게 답할 수 있습니다.

04

예측 불가능성

의도하지 않은 출력이 무대에 나가지 않도록 가드레일이 필요합니다.

Part 06 · Why slow 52 / 55
Conservative by design

무대 시스템은
잘 안 바뀝니다.

Why

안정적인 시스템 = 공연 안정화의 핵심. 새 기술보다 검증된 운영 구조가 우선되는 경우가 많습니다.

But

다양한 인터랙션 실험은 공연 바깥(전시, 이벤트, 미디어아트, 브랜드 경험)에서 활발히 진행 중입니다. 거기서 검증된 것이 무대로 들어옵니다.
